## 第一阶段：静态代码审查与网络源测试（2026-07-21）

### 背景
在空白 VM 跑完整安装前，先对脚本做静态审查、网络源可达性校验和离线包完整性验证。

### 关键发现

#### R1: ms-vscode.cpptools 扩展安装失败（非阻塞）
- **现象**：`setup-cpp-env.ps1` 中 `code --install-extension ms-vscode.cpptools` 安装偶发失败。
- **实际影响**：扩展仍可通过 VS Code 扩展商店手动安装，不影响核心功能。
- **后续修复**：加 90 秒超时 + 静默跳过（DEVLOG Bug #20）。

#### R2: VS Code 中文语言包 locale 配置
- 安装脚本写入 `locale.json` 设为 `zh-cn`，与 VS Code 扩展商店的 `ms-ceintl.vscode-language-pack-zh-hans` 协作生效。

#### R3: Continue 扩展配置 Autodetect 模式
- `config.json`/`config.yaml`/`config.ts` 三件套使用 Autodetect 模式，自动适配本地 Ollama 模型。

#### R4: 用户上下文（Administrator vs {VMUser}）— 根因级别
- **旧入口**（双击 bat 自动 UAC 提权）：脚本以 Administrator 运行，VS Code 扩展装到 `C:\Users\Administrator\.vscode\extensions\`，用户 `{VMUser}` 登录后看不到。
- **修复历程**：曾用 `flashtap-user.txt` 传递上下文 → `-Verb RunAsUser` 降权 → 最终重构为统一"右键以管理员运行"入口（DEVLOG Bug #13/#17/#18）。

#### R5: 桌面快捷方式用户数据目录
- 快捷方式需显式指定 `--user-data-dir` 确保指向正确用户目录。

#### R6: F5 C++ 编译
- **原始状态**：`缺陷修复与复测报告` 标注"F5 C++ 编译未测 | 需 WSL 前提，非 bug"。
- **历史**：旧设计依赖 WSL 提供 g++ 环境，切到 MinGW-w64 后这条链路从未被端到端验证。
- **最终修复**：DEVLOG Bug #21/#23 — 绝对路径 + `type: process`。
